# SDI Manager (SDI-M)

SDI Manager is a comprehensive Network Management System (NMS) designed to monitor and configure network infrastructure using the **NETCONF** protocol. It features a modern, responsive dashboard for real-time visualization of network topology, device status, alarms, and configuration management.

## ğŸš€ Features

*   **Interactive Dashboard**: Real-time overview of network health, traffic statistics, and device distribution.
*   **Topology Visualization**: Dynamic D3.js network graph showing device interconnectivity and status.
*   **Device Management**: CRUD operations for NETCONF-enabled devices (Router, Switch, Firewall, Server).
*   **Automated Data Collection**: 
    *   Background scheduler fetches device data every night at **00:00**.
    *   On-demand fetch capability via the UI.
*   **Alarm Management**: Monitor critical network events and manually acknowledge/dismiss alarms.
*   **Configuration Diff**: Visual comparison between running and startup configurations.
*   **User Authentication**: Secure login, registration, and password management for administrators.
*   **Persistent Storage**: MariaDB integration for long-term storage of device configs, logs, and user data.

## ğŸ“– Usage Guide

### 1. Adding Devices
Navigate to the **Device Manager** tab. Click **"Add Device"** and enter the NETCONF details:
*   **IP Address**: Reachable IP of the network device.
*   **Port**: Default is `830`.
*   **Credentials**: SSH username/password configured on the device.

### 2. Fetching Data
*   **Automatic**: The system automatically attempts to connect to all configured devices at midnight.
*   **Manual**: Click the **"Fetch Data"** button in the top right header to trigger an immediate poll.
    *   *Note: In this demo environment, if actual devices are unreachable, the backend simulates realistic responses for demonstration purposes.*

### 3. Managing Alarms
Navigate to the **Alarms** tab. You can view active alerts generated by unreachable devices or high CPU load. Click the **Trash Icon** to dismiss an alarm once resolved.

## ğŸ”Œ API Reference (Backend)

The Flask backend runs on port `5000`.

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/snapshot` | Returns full topology, device status, and alarms. |
| `GET` | `/api/devices` | List all configured devices. |
| `POST` | `/api/devices` | Add a new device. |
| `DELETE` | `/api/devices` | Remove a device (Query param `?id=...`). |
| `POST` | `/api/fetch` | Trigger manual NETCONF data collection. |
| `DELETE` | `/api/alarms/<id>` | Dismiss a specific alarm. |
| `POST` | `/api/login` | Authenticate user. |
| `POST` | `/api/register` | Create a new user account. |
